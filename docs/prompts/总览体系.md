“每天都能累积”的总览体系：Index + MOCs + Daily + Cards（无插件）

你是我的 Obsidian 知识库架构助手。请输出一个“可长期每天追加”的笔记体系（无插件），并给出可直接落盘的文件模板内容。
输出格式：多文件，每个文件用分隔符：
---FILE: <path>---
<markdown>

目标：
- 我每天学习的内容都能不断整合进同一个体系；
- 允许有一个总览页，但它只做“入口/索引”，避免把正文堆满；
- 每个模块页结构统一（TL;DR / 核心概念 / 常见追问 / 关联链接）；
- 常见追问必须“先简答一句，再给链接”。

请生成这些文件（模板即可）：

1) `Index.md`
- 作为全局入口：只放链接（到各 MOC + 最近 7 天 Daily）
- 给一个“新增内容怎么接入”的 5 步流程（要点式）

2) `MOCs/JWT_MOC.md`
- 主题目录页：列出该主题的模块卡片链接（用 `[[...]]`）
- 给一个 Mermaid flowchart 总览图（图下面再放同样的链接清单，保证可点）

3) `Daily/_Daily_Template.md`
- 每日复盘模板：今日目标/今日证据/今日新增模块/今日更新模块/未解决问题
- 里面必须链接到相关模块卡片（`[[...]]`）

4) `cards/_Module_Template.md`
- 模块页统一结构模板，必须是：
  - TL;DR（3行）
  - 核心概念（要点）
  - 常见追问（2~4条，每条格式：Q/A（1句）→ [[链接]]）
  - 关联链接（至少 3 个 [[...]]）

额外规则：
- 新模块优先放 `cards/`；
- 每天只在 `Daily/YYYY-MM-DD.md` 写“证据/结果/截图说明”，正文知识沉淀到 `cards/`；
- 每次新增模块，都要在对应 `MOCs/*.md` 加链接；并在 `Index.md` 的“最近更新”区加一条。
只输出多文件 Markdown，不要解释过程。
