1) 文档根目录：所有输出文件必须落在 `docs/` 下（如果当前 Obsidian vault 根就是 `<project-root>/docs`，则路径不写 `docs/` 前缀；否则必须写 `docs/xxx`）。
2) 禁止在项目根目录新建 `cards/`、`MOCs/`、`Daily/` 等目录（除非它们位于 `docs/` 下）。
3) 同名文件已存在时：优先“更新/追加”，不要新建重复文件；保持双链不变。
4) 模块页结构固定为：
   - TL;DR（3行）
   - 核心概念（要点）
   - 常见追问（2~4条）：每条必须是「Q：… / A：一句话… / → [[链接]]」
   - 关联链接（至少3个 `[[...]]`）
5) Mermaid：可以使用 `class X internal-link` 做可点击节点，但必须额外提供同样内容的 `[[链接清单]]`（兜底可点）。
